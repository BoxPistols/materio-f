## Overview

We have made the custom radio and custom checkbox components for you to make it easy for you and you don't have to waste your precious time.Please visit [MUI Radio Docs](https://mui.com/material-ui/react-radio-button/) for a proper explanation of the Radio component and [MUI Checkbox Docs](https://mui.com/material-ui/react-checkbox/) for a proper explanation of the Checkbox component.

We have already added these components in the `MUI` components. More informatino on how to create `Themeable component` of MUI can be found [here](https://mui.com/material-ui/guides/themeable-component/#introduction).

## Horizontal Inputs

You can use horizontal inputs as radio or checkbox. You can show title, meta and/or content in the input. 

### Props

```tsx
type CustomInputHorizontalData = {
  value: string // Identify a particular input
  content?: ReactNode // Content of the input
  isSelected?: boolean // If true, the input will be selected at the initial render
} & (
  | {
      meta: ReactNode // Add content to the right side of the title. (To add meta in your input, you need to add the title property as well)
      title: ReactNode // Title for the radio component
    }
  | {
      meta?: never
      title?: never
    }
  | {
      title: ReactNode
      meta?: never
    }
)
type CustomInputHorizontalProps = {
  name: string // Name attribute of the input
  color?: 'primary' | 'secondary' | 'error' | 'warning' | 'info' | 'success' // Color of the selected input
  gridProps?: GridProps // Add props of the MUI's Grid component
  data: CustomInputHorizontalData // Object to create input
} & (
  | {
      type: 'checkbox' // Type of the input
      selected: string[] // Array of selected values
      handleChange: (value: string) => void // Run a function when an option is changed
    }
  | {
      type: 'radio' // Type of the input
      selected: string // Value of the selected radio
      handleChange: (value: string | ChangeEvent<HTMLInputElement>) => void // Run a function when an option is changed
    }
)
```

## Vertical Inputs

You can use vertical inputs as radio or checkbox. You can show title, content and/or asset in the input. In asset you can add an image, an icon or an SVG.

:::caution Warning
If you are passing an SVG as an asset, you need to pass the SVG as a React component.
:::

### Props

```tsx
type CustomInputVerticalData = {
  value: string // Identify a particular input
  title?: ReactNode // Title for the input
  content?: ReactNode // Content of the input
  isSelected?: boolean // If true, the input will be selected at the initial render
  asset?: ReactNode // Render an asset in the input like an image or an icon or svg
}
type CustomInputVerticalProps = {
  name: string // Name attribute of the input
  color?: 'primary' | 'secondary' | 'error' | 'warning' | 'info' | 'success' // Color of the selected input
  gridProps?: GridProps // Add props of the MUI's Grid component
  data: CustomInputVerticalData // Object to create input
} & ({
  type: 'checkbox' // Type of the input
  selected: string[] // Array of selected values
  handleChange: (value: string) => void // Run a function when an option is changed
} | {
  type: 'radio' // Type of the input
  selected: string // Value of the selected radio
  handleChange: (value: string | ChangeEvent<HTMLInputElement>) => void // Run a function when an option is changed
})
```

## How to override components

To override or customized the components, follow these steps:

Let us say you want to customize the `CustomInputHorizontal` component.(you can customize other custom input components as well by following the same steps).

1. You need to create a new file (you can name the file whichever you want; let us say that you have named `src/components/custom-inputs/UserHorizontal.tsx` file). Copy the code from `src/@core/components/custom-inputs/horizontal.tsx` file, paste it into the new file. Also copy the related types from `src/@core/components/custom-inputs/types.ts` file and paste it into the `src/components/custom-inputs/types.ts` file.  
2. Now, you can make changes as per your requirements in the `UserHorizontal.tsx` file.
3. We have already added the `CustomInputHorizontal` component in the `MUI` components. You may refer to the `src/compponents/theme/type.ts` file on how to add a custom component in the `MUI` components. 

That's it! Now you can import the `UserHorizontal` component in your project and use it.

## How to override styles 

To override or customized the styles, follow these steps:

Let us say you want to customize the `CustomInputHorizontal` component.(you can customize other custom input components as well by following the same steps).

You can override the styles in both the `src/compoonents/theme/Theme.ts` or `src/components/theme/MergedTheme.ts` file.

You may refer to the following code snippet:
